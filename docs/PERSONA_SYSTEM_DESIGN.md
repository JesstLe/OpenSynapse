# OpenSynapse 多重人格（Persona）与自定义 Agent 系统设计文档

## 1. 概述
本系统旨在打破“单一导师”模式，通过引入 **Persona（人格）** 体系，使 OpenSynapse 能够胜任不同学科的辅导任务。系统支持预设专家人格、隐藏私密人格以及用户自定义 Agent。

## 2. 核心架构

### 2.1 Persona 数据结构
所有人格对象遵循以下接口定义：

```typescript
export interface Persona {
  id: string;           // 唯一标识
  name: string;         // 昵称（如：逻辑法兰克）
  icon: string;         // Lucide 图标名
  description: string;  // 功能简介
  systemPrompt: string; // 注入给 LLM 的核心指令
  category: 'cs' | 'math' | 'law' | 'finance' | 'custom' | 'hidden';
  isLocked: boolean;    // 系统内置，禁止用户删除/修改
  isHidden: boolean;    // 是否为隐藏状态（需通过秘籍解锁）
}
```

### 2.2 隐藏与加密机制
- **进入方式**：在 App Header 的 Logo 处执行“七连击”（7 clicks within 2s）。
- **反馈**：Logo 产生一阵微弱的紫色脉冲动画，随后在人格列表中出现“隐藏人格”或解锁“自定义 Agent”高级编辑模式。
- **存储加密**：隐藏人格的 Prompt 在代码中以 Base64 存储，在运行时解密。
- **持久化**：自定义人格存储于 Firestore `custom_personas` 集合，与用户 UID 绑定。

## 3. 预设人格定义（考研向）

### 3.1 计算机科学导师（锁定）
- **Prompt**：完全保留现有“发生认识论”与“痛点驱动”框架。代码默认 C++。
- **目标**：底层原理、工程实践。内容严禁更改。

### 3.2 考研数学专家（新增）
- **核心逻辑**：强调概念的几何直观与代数严谨性。
- **输出规范**：
    - 使用 $\LaTeX$ 渲染公式。
    - 针对考研重难点（如中值定理、线性空间），先讲“为什么要定义它”，再讲“它解决了什么几何/计算问题”。
- **语气**：严密、冷静、富有启发性。

### 3.3 法学研学导师（新增）
- **核心逻辑**：IRAC (Issue, Rule, Application, Conclusion) 结构化输出。
- **输出规范**：
    - 引用法条必须标注来源（如：《民法典》第xxx条）。
    - 强调法律关系的梳理。
- **语气**：中立、客观、逻辑严丝合缝。

### 3.4 金融/经管专家（新增）
- **核心逻辑**：供需模型、权衡分析 (Trade-off)、宏观背景。
- **输出规范**：
    - 涉及概念时提供真实市场案例。
    - 强调时效性与理论模型的适用边界。

## 4. 自定义 Agent 手册

用户可以在“人格实验室”创建新的人格，系统会自动将其注入到 Chat 循环中。

### 4.1 编写建议
- **角色设定**：明确告知 AI “你是谁”。
- **任务目标**：明确告知 AI “你要做什么”。
- **约束条件**：明确告知 AI “你不能做什么”或“必须遵循什么格式”。

## 5. 交互界面设计

### 5.1 聊天视图
在消息列表顶部增加一个水平滑动的标签栏（Pills），点击瞬间切换当前对话的人格上下文。

### 5.2 设置视图
新增“人格实验室”子页面，提供卡片式的人格管理界面：
- **系统卡片**：只读，显示“已锁定”图标。
- **自定义卡片**：支持编辑 Prompt 和删除。
