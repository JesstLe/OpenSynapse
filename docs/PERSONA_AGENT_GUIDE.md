# OpenSynapse 导师人格 (Persona) & Agent 开发攻略

OpenSynapse 的“人格实验室”允许你定义各种具备专业深度、特定语调或教学风格的专家级导师。这些导师不仅拥有独特的交互界面（图标与颜色），更核心的是其背后的 **System Instruction (系统指令)**。

## 1. 什么是系统指令 (System Prompt)？

系统指令是 AI 的“灵魂”。它定义了 AI 在整个对话周期中的：
- **身份定位** (如：资深数学建模竞赛评委)
- **专业边界** (如：仅讨论法考相关的行政法问题)
- **输出风格** (如：使用苏格拉底式提问引导学生，而不是直接给答案)
- **特定模版** (如：每条回复必须包含一个 LaTeX 数学公式)

---

## 2. 如何编写高质量的人格提示词？

建议遵循 **Identity - Task - Constraint (ITC)** 框架：

### A. 身份 (Identity)
不要只说“你是一个数学老师”，要更具体：
> “你是一位有 20 年经验的中国考研数学特级教师。你擅长解析抽象的线性代数概念，并能从历年真题的角度分析知识点的重要性。”

### B. 任务 (Task)
明确 AI 的核心工作：
> “你的任务是帮助学生掌握高等数学的核心定理。你会先询问学生的理解程度，然后通过分步骤推理 (Chain of Thought) 来拆解复杂题目。”

### C. 约束 (Constraint)
防止 AI 跑题或偷懒：
- **严禁直接给答案**：要求 AI 仅提供思路或针对错误步骤进行点评。
- **强制使用 LaTeX**：要求所有数学变量、公式必须放在 `$...$` 或 `$$...$$` 中。
- **引用知识库**：本项目支持 RAG（检索增强生成），AI 会自动获取你的笔记内容。你可以在 Prompt 中加入：“请优先结合用户知识库中的已知笔记来解释新概念。”

## 3. 实战案例：考研导师系列

### 案例 1：数学导师 (Sigma)
**系统指令示例：**
```text
你是一位深耕考研数学的顶级导师。
1. 你的所有回复必须逻辑严密。凡涉及变量、函数、导数、积分等数学表达，必须使用 LaTeX 格式。
2. 当用户提出一个概念时，请先给出它的几何意义，再给出代数定义。
3. 如果用户发来一道题目，严禁直接给完整解答。请先分析考点，然后询问用户在哪个步骤遇到了卡点。
```

### 案例 2：法学导师 (Gavel)
**系统指令示例：**
```text
你是一位针对中国法考（法律职业资格考试）的专业导师。
1. 风格：冷静、权威、用词考究。
2. 逻辑：对于每一个法律问题，遵循“法条检索 -> 案情拆解 -> 构成要件分析 -> 结论总结”的 IRAC 模式。
3. 重点：强调法理背后的立法目的，帮助学生在理解的基础上记忆。
```

## 4. 隐藏人格：计算机导师

*提示：OpenSynapse 的核心人格——计算机导师，需要通过**快速点击应用 Logo 7 次**（500ms 内）来解锁。Logo 会发出紫色脉冲光效作为反馈。*

**计算机导师特点：**
- 深耕底层原理，用"发生认识论"带你拆解复杂工程
- 核心教学理念：知识不是凭空产生的，而是为了解决特定"痛点"而发明的
- 严格遵循"三部曲"教学：史前时代 → 笨办法 → 救世主登场
- 默认使用 C++ 作为编程教学语言

## 5. 提示词进阶：角色扮演 vs 技能赋予
- **角色扮演**：更看重语气，适合语言学习（如：雅思口语考官）。
- **技能赋予**：更看重流程，适合逻辑工作（如：Python 重构专家，要求每次回复必须先写出重构计划，再展示 Diff）。
